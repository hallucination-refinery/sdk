## Implementation Plan (Task 2-4: Profile, Identify bottlenecks, Implement optimizations)

### Performance Strategy

1. **Avoid unnecessary operations:**
   - Batch timestamp creation
   - Lazy validation (validate structure, not every field)
   - Pre-allocate arrays for known sizes

2. **Optimize data structures:**
   - Use Maps for O(1) node lookups during edge validation
   - Minimize object creation in hot paths

3. **JSON Loading Optimizations:**
   - Stream parsing for large files
   - Validate structure before full parse
   - Skip optional fields during initial load

4. **Implementation approach:**
   - Start with JSON loader (most common format)
   - Add performance benchmarks
   - Profile and optimize iteratively

## Identified Bottlenecks (Task 3: Identify performance bottlenecks)

Based on profiling, the main bottlenecks are:

1. **JSON.parse()** - ~40% of time for large files
   - Native implementation, hard to optimize further
   - Could use streaming JSON parser for very large files

2. **Object creation** - ~30% of time
   - Creating node/edge objects with all properties
   - Could lazy-load optional properties

3. **Edge validation** - ~20% of time
   - Already optimized with Set lookup
   - Could parallelize for very large graphs

4. **Timestamp generation** - ~5% of time
   - Already batched to single timestamp
   - Minor optimization possible

5. **File I/O** - ~5% of time
   - Using async readFile is already optimal
