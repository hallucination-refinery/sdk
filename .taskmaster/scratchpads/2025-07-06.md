## Design Phase

### NodeSprite Component Requirements
Based on the architecture and current implementation:

1. **Location**: `packages/canvas-r3f/src/components/NodeSprite.tsx`
2. **Purpose**: Generic, reusable sprite component for displaying node labels/text
3. **Features**:
   - Text rendering capability using Three.js sprites
   - Configurable text, color, size, and position
   - Billboard behavior (always faces camera)
   - Performance optimized for 1000+ nodes
   - Follows r3f patterns and conventions

### API Design
```typescript
interface NodeSpriteProps {
  text: string
  position?: [number, number, number]
  scale?: [number, number, number] | number
  color?: string
  fontSize?: number
  fontFamily?: string
  backgroundColor?: string
  backgroundOpacity?: number
  padding?: number
  visible?: boolean
  opacity?: number
  anchorX?: number | 'left' | 'center' | 'right'
  anchorY?: number | 'top' | 'middle' | 'bottom'
}
```

### Implementation Strategy
1. Create a canvas-based text sprite using Three.js CanvasTexture
2. Implement dynamic text rendering with proper sizing
3. Add background support for better readability
4. Ensure proper disposal of textures to prevent memory leaks
5. Make it compatible with the existing Node component structure

### Architecture Alignment
- Fits within Layer 4 (Canvas) as per the six-layer model
- No dependencies outside of React, Three.js, and r3f
- Follows the principle of hiding implementation details
- Can be commanded via the existing renderer command pattern

### Implementation Details
- Used Three.js CanvasTexture for text rendering
- Implemented dynamic canvas sizing with power-of-2 dimensions for GPU optimization
- Added configurable text styling (color, font, size)
- Included background support with opacity for better readability
- Proper texture disposal to prevent memory leaks
- Billboard behavior is automatic with Three.js sprites
- Integrated seamlessly with existing Node component

### Performance Considerations
- Canvas dimensions are rounded to power-of-2 for better GPU performance
- Texture is created once per text change using React's useMemo
- Proper cleanup on unmount
- Depth write disabled for proper transparency handling
