### Widget Package Build Errors (Task 2)

Both widget packages have build scripts replaced with `echo 'widget build skipped'` due to:

**Common Issues**:
- Unused imports and variables (strict TypeScript settings)
- Missing type definitions for test utilities (vi from Vitest)
- Type compatibility issues (null vs undefined)
- Incomplete implementations with unused props/parameters

**widget-aperture specific**:
- Unused React imports and hooks
- Edge type issues
- Multiple unused declarations

**widget-hud specific**:
- CSS property type issues (WritingMode)
- Similar unused import problems

**Root Cause**: Strict TypeScript settings (`noUnusedLocals`, `noUnusedParameters`) combined with incomplete implementations.

### Ops Package tsc-b Investigation (Task 3)

**Finding**: The ops package build was changed from `"tsup && tsc -b"` to just `"tsup"` in commit 09ef7b45.

**Details**:
- Original build script: `"build": "tsup && tsc -b"`
- Current build script: `"build": "tsup"`
- tsup.config.ts has `dts: false` (not generating type definitions)
- Change was made in "baseline-green" commit to stub types and skip failing builds

**Implication**: The TypeScript type generation step was removed to achieve a passing build, but this means the package may not be generating proper type definitions.
