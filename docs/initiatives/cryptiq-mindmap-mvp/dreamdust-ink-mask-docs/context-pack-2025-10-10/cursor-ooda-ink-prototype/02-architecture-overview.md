## Data & Uniforms Flow (End-to-End)
- Pointer movement is normalized in `InkSurface`, which pushes delta samples (`onForceSample`) and splat impulses (`onForceSplat`) after clamping UVs and brush radius ([apps/cryptiq-mindmap-demo/app/components/dreamdust/InkSurface.tsx:296](apps/cryptiq-mindmap-demo/app/components/dreamdust/InkSurface.tsx:296), [apps/cryptiq-mindmap-demo/app/components/dreamdust/InkSurface.tsx:389](apps/cryptiq-mindmap-demo/app/components/dreamdust/InkSurface.tsx:389)).
- The stage routes `onForceSample` into `applyTempForce`, updating `uTempForce`, `uTempIntensity`, and `uTempCenter` for the Dreamdust material, while `onForceSplat` forwards taps to `FluidSim.addForce` ([apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:1568](apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:1568), [apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:3214](apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:3214)).
- `FluidSim.addForce` writes the splat into the write target and swaps the velocity ping-pong pair; `step` then runs advect â†’ divergence â†’ Jacobi (Ă—iterations) â†’ projection per frame before exposing the latest velocity texture ([apps/cryptiq-mindmap-demo/app/components/dreamdust/fluid/FluidSim.ts:191](apps/cryptiq-mindmap-demo/app/components/dreamdust/fluid/FluidSim.ts:191), [apps/cryptiq-mindmap-demo/app/components/dreamdust/fluid/FluidSim.ts:206](apps/cryptiq-mindmap-demo/app/components/dreamdust/fluid/FluidSim.ts:206)).
- `FluidDriver` executes with `useFrame` priority 1, calling `sim.step(delta)` and keeping `uVelocity`, `uVelTexInvSize`, `uVelToNdc`, and `uInkBlend` live on both Dreamdust materials each frame ([apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:900](apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:900), [apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:915](apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:915)).
- The init effect seeds uniforms so `DreamdustMaterial` has a valid velocity texture from the first frame, leveraging the 1Ă—1 dummy texture only during bootstrap ([apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:1383](apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:1383), [apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:99](apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:99)).
- In the vertex shader, Dreamdust samples `uVelocity` using clip-space UVs, multiplies by `uVelToNdc`, and mixes displacement based on `uInkBlend`; temporaries (`uTempForce`, `uTempIntensity`, `uTempRadius`) modulate additional screen-space pushes ([apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:410](apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:410), [apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:520](apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:520)).
- Fragment shading blends ink tint and alpha using `uInkTex`, cascade controls, and premultiplied alpha output, matching the additive or normal blend mode selected per preset ([apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:552](apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:552), [apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:770](apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:770)).

## Rendering Pipeline Stages
- CPU frame hooks: `TempForceDriver` decays temporary forces each frame, `FluidDriver` advances the fluid solver before draw submission, and `SimDriver` (when enabled) updates particle textures ([apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:104](apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:104), [apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:900](apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:900), [apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:3290](apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:3290)).
- GPU simulation passes per frame: add-force when splats occur, advect into velocity write target, compute divergence, run Jacobi relaxation `FLUID_JACOBI_ITERS` times, project to divergence-free velocity, then expose the read target for rendering ([apps/cryptiq-mindmap-demo/app/components/dreamdust/fluid/FluidSim.ts:191](apps/cryptiq-mindmap-demo/app/components/dreamdust/fluid/FluidSim.ts:191), [apps/cryptiq-mindmap-demo/app/components/dreamdust/fluid/FluidSim.ts:206](apps/cryptiq-mindmap-demo/app/components/dreamdust/fluid/FluidSim.ts:206)).
- Final draw: Dreamdust `points` geometry renders with `ShaderMaterial` displacement applied after clip coordinates, binding the latest velocity texture and point sprite uniforms before Three dispatches the draw call ([apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:3320](apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:3320), [apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:520](apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:520)).

## Shader/Material Integration (GLSL3)
- Both the fluid passes and Dreamdust material rely on Threeâ€™s GLSL3 pipeline (`ShaderMaterial` with `glslVersion`), avoiding manual `#version` statements while still enabling WebGL2-only syntax ([apps/cryptiq-mindmap-demo/app/components/dreamdust/fluid/FluidSim.ts:151](apps/cryptiq-mindmap-demo/app/components/dreamdust/fluid/FluidSim.ts:151), [apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:748](apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:748)).
- `DreamdustMaterial` seeds every uniform (including a 1Ă—1 float `uVelocity`) so shader programs link before the sim spins up; uniform aliasing keeps legacy names mapped to new keys ([apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:99](apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:99), [apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:706](apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:706)).
- `USE_VERTEX_INK`, `USE_GAUSSIAN`, and other defines are toggled per material instance, but the displacement path is gated solely by `uVelToNdc`/`uInkBlend`, so a zero scale disables the fluid flow without recompiling shaders ([apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:726](apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:726), [apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:520](apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:520)).

## Architectural Choices, Gaps, Risks (with mitigations)
- **Screen-space displacement** â€“ Vertex positions are nudged after projection using `uVelocity`, keeping under-finger motion aligned with the screen while allowing `uInkBlend` to dial in the mix ([apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:520](apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:520)). Mitigation for over-displacement is to cap `uVelToNdc` via stage tunables.
- **GLSL3 ShaderMaterial** â€“ Relying on Threeâ€™s ShaderMaterial with `glslVersion` avoids RawShaderMaterial boilerplate but assumes all attributes/uniforms follow Three conventions; upgrades must audit defines to prevent preamble clashes ([apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:748](apps/cryptiq-mindmap-demo/app/components/dreamdust/DreamdustMaterial.ts:748)).
- **Uniform single source** â€“ Stage primes velocity uniforms once and suppresses effect deps to avoid TDZ, but material recreation (e.g., preset swap) can miss a refresh; add a material lifecycle hook to reapply textures when remounting ([apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:1383](apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:1383), [apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:1420](apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:1420)).
- **Platform risk** â€“ Without WebGL2 (or EXT_color_buffer_float for float RTTs) the constructor throws; we currently catch and log but lack a degraded ink path. Consider detecting caps earlier and disabling the fluid overlay gracefully ([apps/cryptiq-mindmap-demo/app/components/dreamdust/fluid/FluidSim.ts:82](apps/cryptiq-mindmap-demo/app/components/dreamdust/fluid/FluidSim.ts:82), [apps/cryptiq-mindmap-demo/app/components/dreamdust/fluid/FluidSim.ts:107](apps/cryptiq-mindmap-demo/app/components/dreamdust/fluid/FluidSim.ts:107)).
- **Visibility gap** â€“ Latest smoke shows clean init but no visible particles, so validation hinges on boosting displacement/alpha via `fluidBoost` and ink intensity knobs before tightening budgets ([docs/initiatives/cryptiq-mindmap-mvp/dreamdust-ink-mask-docs/context-pack-2025-10-10/cursor-ooda-ink-prototype/10-latest-smoke-evidence.md:9](docs/initiatives/cryptiq-mindmap-mvp/dreamdust-ink-mask-docs/context-pack-2025-10-10/cursor-ooda-ink-prototype/10-latest-smoke-evidence.md:9), [apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:1222](apps/cryptiq-mindmap-demo/app/components/PointCloudStage.tsx:1222)).
